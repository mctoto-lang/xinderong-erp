项目名称：塑料再生材料网页 ERP 管理系统

一、项目定位
请基于以下结构生成一个中文后台网页项目，类型为“生产制造 + 采购销售 + 库存财务一体化 ERP 系统”。系统面向塑料再生材料企业，业务涵盖采购、出货、生产加工、库存、财务、客户、供应商、物流、图表分析、系统管理。整体风格要求专业、简洁、稳重、信息密度高，适合 PC 端后台管理系统，也要兼顾平板和移动端的基础适配。

二、整体网页结构
1. 页面模式
这是一个“登录页 + 登录后的后台单页应用”结构。

2. 登录后整体布局
- 左侧为固定侧边栏导航。
- 右侧为主内容区域。
- 主内容区顶部有细高页头，包含：
  当前模块标题、侧边栏收起按钮、通知铃铛。
- 主内容区主体为滚动内容区，不同模块在此切换显示。

3. 侧边栏结构
- 顶部显示企业 Logo 区与公司名称、副标题。
- 中部是模块导航菜单。
- 底部是当前登录用户信息卡片，下拉后可退出登录。

4. 导航模块
- 首页仪表盘
- 进货管理
- 出货管理
- 生产加工
- 库存管理
- 财务管理
- 客户管理
- 供应商管理
- 物流运输
- 图表分析
- 系统管理

5. 数据库配置要求
- 使用 Neon PostgreSQL 作为数据库（云托管 PostgreSQL 服务）。
- ORM 层使用 Prisma，通过 DATABASE_URL 环境变量连接数据库。
- 连接字符串格式：postgresql://user:password@host/database?sslmode=require
- 必须启用 SSL 连接（sslmode=require），因为 Neon 要求加密连接。
- 数据库连接池使用 Neon 提供的 pooler 端口，支持高并发访问。
- Prisma schema 中所有模型需映射到 PostgreSQL 数据类型（如 Json、Decimal、DateTime 等）。
- 支持数据库迁移（prisma migrate），初始部署时自动执行 npx prisma db push 创建表结构。

6. 全局交互要求
- 所有列表页支持空状态、加载骨架屏、分页。
- 常见操作使用弹窗 Dialog、右侧抽屉 Sheet、确认框 AlertDialog。
- 所有金额统一显示为人民币，保留两位小数。
- 所有重量统一使用 KG。
- 所有日期选择统一使用日历选择器。
- 操作结果使用 Toast 提示。
- 需要支持导出 Excel。
- 进货单、出货单支持打印预览。

三、视觉与风格提示
- 整体采用现代化后台风格，浅色主题。
- 卡片、表格、标签、统计面板层级明确。
- 重视数据可读性，适合财务、库存、生产管理场景。
- 状态颜色要明确：
  待处理用灰/黄，
  进行中用蓝，
  成功完成用绿，
  风险预警或未付款用红，
  部分付款/部分回款用橙。
- 表格要整洁，支持数字右对齐、编码用等宽字体、标签状态用 Badge。

四、页面详细生成提示词

【1. 登录页】
页面用途：系统登录入口。
页面布局：
- 居中大卡片布局，卡片宽度偏大，左右分栏。
- 左侧为登录表单。
- 右侧为企业相关宣传图或背景图，仅桌面端显示。
- 卡片下方显示服务条款和隐私政策说明。
表单内容：
- 标题：欢迎回来。
- 副标题：公司名称。
- 用户名输入框。
- 密码输入框。
- 忘记密码入口，使用悬浮说明卡。
- 登录按钮，带 loading 状态。
- 第三方登录入口图标：微信、QQ、手机。
- 注册入口，但可提示“暂未开放，仅内部使用”。
交互要求：
- 登录失败显示红色错误提示条。
- 登录按钮提交时显示转圈状态。
- 整体气质偏企业内部系统，不要做成互联网营销风。

六、重点业务逻辑提示
- 进货会增加原料库存。
- 生产开工会锁定或消耗原料。
- 生产完工会增加成品库存，并计算成品率。
- 出货会扣减成品库存，并产生应收。
- 进货付款会减少应付。
- 出货回款会减少应收。
- 财务页要能从进货和出货业务中汇总出应付、应收、收支流水和利润。
- 库存页要能追踪每一次库存变化的来源。

七、数据库表单加载失败处理与手动拉取
当页面因网络问题、数据库连接异常或表尚未创建等原因，无法成功获取到数据库表单数据时，必须进行以下错误处理和交互设计：

1. 错误提示展示
- 当数据加载失败时，页面主内容区域显示一个醒目的错误状态面板，而非空白页或浏览器默认错误。
- 错误面板包含以下元素：
  - 错误图标（如 AlertTriangle 或 DatabaseX 图标），使用醒目的橙红色。
  - 错误标题："数据加载失败" 或 "无法连接到数据库"。
  - 错误描述：根据错误类型显示具体原因，例如：
    · "网络连接异常，请检查网络后重试。"
    · "数据库表尚未创建，请先执行数据库初始化。"
    · "数据库连接超时，请稍后再试。"
    · "权限不足，无法访问该数据表。"
  - 错误详情折叠区：用户可展开查看完整的错误信息和技术细节（如请求 URL、状态码、错误堆栈），方便排查问题。

2. 手动拉取数据库表单按钮
- 错误面板底部提供两个操作按钮：
  - 「重新加载」按钮（主按钮，蓝色）：重新发起数据请求，尝试重新加载当前页面数据。
  - 「手动同步数据库表结构」按钮（次按钮，带图标）：点击后调用一个专门的 API 端点（如 /api/sync-db），该端点执行以下操作：
    · 检测数据库连接状态。
    · 执行 prisma db push 或 prisma migrate deploy，确保数据库表结构与 Prisma schema 同步。
    · 返回同步结果（成功/失败/已有最新结构）。
    · 成功后自动重新加载页面数据。
  - 同步按钮在执行期间显示 loading 状态（旋转图标 + "正在同步..." 文字），防止重复点击。
  - 同步完成后显示 Toast 提示：
    · 成功："数据库表结构已同步，数据已重新加载。"
    · 失败："同步失败：{具体错误信息}，请联系管理员。"

3. 自动重试机制
- 数据加载失败后，页面自动每 10 秒重试一次（最多 3 次），并在界面上显示倒计时提示（如 "将在 10 秒后自动重试..."）。
- 用户可随时点击「重新加载」手动触发重试，重置自动重试计数器。
- 如果自动重试全部失败，则稳定显示错误面板，不再自动重试。

4. 全局拦截
- 所有 API 请求使用统一的错误拦截器，当后端返回数据库相关错误（如表不存在、连接失败）时，自动跳转到错误面板状态，而非在每个页面单独处理。
- 错误面板设计风格与系统整体风格一致，使用卡片包裹，居中或填充整个内容区域。
